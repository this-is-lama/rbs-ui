# RBS UI

Frontend-приложение для сервиса онлайн-бронирования столов в ресторанах.

Стек: React 19, TypeScript 5.9, Vite 8, React Router 7, SCSS Modules.

## О проекте
RBS UI — это SPA для пользователей и менеджеров ресторанов.

Приложение позволяет:

- искать рестораны и фильтровать их по категориям;
- открывать страницу ресторана с галереей, меню и схемой зала;
- бронировать стол по дате и временному интервалу;
- просматривать блюда и добавлять их в заказ;
- авторизовываться, управлять профилем и бронированиями;
- работать с менеджерским кабинетом: создавать рестораны, редактировать блюда, схему зала, фотографии и список броней.

## Основные возможности
### Для гостя и пользователя
- главная страница с презентацией сервиса;
- каталог ресторанов с фильтрами и категориями;
- страница ресторана с меню, схемой зала и модальным окном бронирования;
- страница блюда;
- корзина бронирования и заказа;
- регистрация, вход и профиль пользователя;
- переключение языка `ru/en`.

### Для менеджера и администратора
- раздел `Мои рестораны`;
- создание и редактирование ресторана;
- управление блюдами ресторана;
- управление схемой зала и столами;
- просмотр бронирований ресторана;
- загрузка и удаление фотографий ресторана и блюд.

## Быстрый старт
### 1. Установить зависимости
```bash
npm install
```

### 2. Проверить адрес backend
Клиент использует базовый адрес API из `src/shared/api/api-client.ts`.

По умолчанию это:

```env
API_URL=http://localhost:8080
```

В репозитории уже есть файл `.env`, но если backend запущен на другом адресе, приведи значение к нужному окружению.

### 3. Запустить проект
```bash
npm run dev
```

### 4. Проверить production-сборку
```bash
npm run build
```

## Структура проекта
Проект организован в FSD-подобной структуре:

```text
src/
├─ app/        # инициализация приложения, провайдеры, роутинг, глобальные стили
├─ pages/      # route-level страницы
├─ widgets/    # крупные секции страниц и составные блоки
├─ features/   # пользовательские сценарии и бизнес-действия
├─ entities/   # сущности домена: restaurant, booking, user, dish
└─ shared/     # переиспользуемые ui, lib, api, config, styles
```

### Ключевые директории
- `src/app/providers` — auth, language, router;
- `src/entities/restaurant` — типы, API и карточки ресторанов/блюд;
- `src/widgets/restaurant-details` — страница ресторана, меню, схема зала, бронирование;
- `src/widgets/my-restaurants` — кабинет менеджера;
- `src/shared/api` — HTTP-клиент и базовые API-типы;
- `src/shared/config/routes` — централизованные пути приложения.

## Архитектурные детали
- Используется алиас `@` на `src`.
- Авторизация построена через `AuthProvider` и `tokenStorage`.
- Axios-клиент поддерживает автоматическое обновление `accessToken` через `refreshToken`.
- Часть пользовательских данных хранится в `localStorage`:
  - токены авторизации;
  - корзина бронирования;
  - корзина блюд;
  - выбранный язык интерфейса.
- Для страниц и переключения категорий уже встроены плавные анимации переходов.
